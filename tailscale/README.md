# Tailscale Exit Node для OpenWrt

Однострочный скрипт для автоматической установки и настройки Tailscale на OpenWrt 25.12 (с пакетным менеджером `apk`) в режиме **Exit Node**.  
Скрипт **автоматически определяет локальную подсеть** и рекламирует её, поэтому все устройства в вашей локальной сети становятся доступны через Tailscale.

---

## 🚀 Быстрая установка

Подключитесь к роутеру по SSH и выполните:

```bash
sh <(wget -O - https://raw.githubusercontent.com/theodorez3non/ru-routing/main/tailscale/install.sh)
```

Скрипт автоматически:
- Установит Tailscale (через `apk` или скачиванием бинарника, если мало места)
- Определит подсеть вашей локальной сети (из интерфейса `lan`) и добавит её в `--advertise-routes`
- Настроит сетевой интерфейс `tailscale0`
- Настроит файрвол (зона `tailscale`, forwarding в `lan` и `wan`, правила для SSH и веб-интерфейса)
- Включит IP-форвардинг
- Запустит Tailscale с параметрами `--advertise-exit-node --ssh --netfilter-mode=off` и автоматически определённой подсетью
- Добавит автозапуск туннеля после перезагрузки (в `/etc/rc.local`)

После установки скрипт выведет ссылку для авторизации. Перейдите по ней, войдите в свой аккаунт Tailscale, затем в веб-админке Tailscale включите для этого устройства опцию **Exit node**.

---

## 🔧 Требования

- **OpenWrt 24.10 или 25.12** (с поддержкой `apk`)
- **Архитектура**: `mips`, `armv7l`, `aarch64`, `x86_64`
- **Оперативная память**: от 60 МБ свободной
- **Постоянная память**: от 25 МБ свободной (если меньше — скрипт установит во `/tmp`)
- **Права root**
- Интерфейс `lan` должен быть настроен (по умолчанию он есть на всех роутерах)

---

## 📋 Что делает скрипт

| Действие | Описание |
|----------|----------|
| Установка | Через `apk` (если есть место) или скачиванием бинарника с официального сайта Tailscale |
| Определение подсети | Автоматически вычисляет сеть интерфейса `lan` (например, `192.168.1.0/24`) с помощью `ipcalc.sh` или `ip addr` |
| Сеть | Создаёт интерфейс `tailscale0` (protocol `unmanaged`) |
| Файрвол | Создаёт зону `tailscale` с `masq=1`, forwarding в `lan` и `wan`, открывает порты 22, 80, 443 |
| IP-форвардинг | Включает `net.ipv4.ip_forward` |
| Запуск | `tailscale up --accept-dns=false --advertise-exit-node --ssh --netfilter-mode=off --advertise-routes=<авто-подсеть>` |
| Автозапуск | Добавляет команду в `/etc/rc.local` — туннель поднимается после каждой перезагрузки |

---

## 🖥️ Использование после установки

1. **Авторизуйтесь** по ссылке, которую выведет скрипт.
2. **Включите Exit Node** в админке Tailscale:  
   Зайдите в `https://login.tailscale.com/admin/machines`, найдите свой роутер и включите переключатель **Exit node**.
3. **Подключитесь** к роутеру по Tailscale IP:
   - Веб-интерфейс: `http://<Tailscale_IP>`
   - SSH: `ssh root@<Tailscale_IP>`
   - Также станут доступны все устройства в локальной сети (по их локальным IP) через роутер.

---

## 🗑️ Деинсталляция

Если нужно полностью удалить Tailscale и все настройки, выполните:

```bash
sh <(wget -O - https://raw.githubusercontent.com/theodorez3non/ru-routing/main/tailscale/remove.sh)
```

Скрипт удалит:
- Пакет Tailscale (через `apk`)
- Бинарные файлы
- Конфигурацию (`/var/lib/tailscale`, `/etc/tailscale`)
- Сетевой интерфейс `tailscale`
- Все правила и зону файрвола, связанные с `tailscale`
- Строку автозапуска из `/etc/rc.local`
- Init-скрипт `/etc/init.d/tailscale`

## ⚠️ Примечания

- Если у вас используется **nftables** (по умолчанию в OpenWrt 25.12), скрипт запускает Tailscale с `--netfilter-mode=off`, чтобы избежать конфликтов. Управление файрволом полностью ложится на UCI.
- **Подсеть определяется автоматически** — вам не нужно указывать её вручную. Если вы хотите изменить рекламируемые маршруты, после установки выполните `tailscale down` и `tailscale up ...` с нужными параметрами.
- Скрипт проверяет свободное место в `/` и, если его меньше 25 МБ, устанавливает Tailscale во временную директорию `/tmp`. После перезагрузки установка будет потеряна, но скрипт автоматически переустановит её при следующем запуске (если вы снова выполните команду установки).
